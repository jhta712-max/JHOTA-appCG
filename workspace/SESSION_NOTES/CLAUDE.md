# SESSION_NOTES/ — Reglas locales

## Qué va aquí
Notas de cada sesión de trabajo: decisiones, problemas encontrados, soluciones implementadas, y aprendizajes.

## Convenciones

**Nombre de archivos:**
- `SESSION_YYYY-MM-DD.md` — Una sesión por archivo
- `KEY_DECISIONS.md` — Decisiones arquitectónicas importantes
- `LEARNINGS.md` — Cosas aprendidas que no se olviden

**Estructura de contenido:**
```markdown
# Sesión YYYY-MM-DD — Tema principal

## Resumen ejecutivo
- Qué se hizo
- Resultados
- Problemas encontrados

## El problema
Descripción breve del qué/por qué

## Soluciones implementadas
1. [Solución A]
2. [Solución B]

## Estado final
✅ Qué funciona ahora

## Aprendizajes
Lecciones para futuras sesiones

## Próximos pasos
Qué hacer después
```

## NO toques
- Archivos de sesiones pasadas (son histórico)
- Números o fechas — si necesitas actualizar, crea NUEVO archivo

## Objetivo
Que la próxima sesión entienda qué pasó sin preguntar.
